## 1. TBC ..

When someone reports a bug with an instruction to reproduce the problem, but you can't actually reproduce it on your dev environment, it doesn't mean it's a user error. You still have a problem with your code that you need to fix. I see time and time again when a developer demands the user to prove that his/her code is broken before they even start investigating the issue; as if it's user's responsibility to make developer's life easier. If the software is an open-source and developer only works on a volunteer basis, you should clearly state at the top of the README that you have no intention of helping user to troubleshoot their problems - and perhaps point users to a better alternative to using your application.
